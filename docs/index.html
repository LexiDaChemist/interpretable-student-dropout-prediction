Predicts whether a student is likely to Dropout, stay Enrolled, or Graduate

A transparent multiclass logistic regression model, with clear visuals, evaluation, and simple scenario tests.

Project Overview

This project predicts student outcomes (Dropout, Enrolled, Graduate) using academic progress, financial indicators, demographic information, and economic context. The main focus is interpretability: the goal is to understand which factors push predictions up or down, not just to get a score.

I chose this topic because dropout is a real problem for students and institutions. A model like this can support early outreach (advising, tutoring, financial support) when used responsibly.

Learning Model Used

Logistic regression

A multiclass logistic regression model because it estimates the probability of each outcome class using a weighted combination of input features. This makes it a strong choice for interpretability because data can be inspected by each of the learned weights to explain the predictions.

Why not a more complex model? Models like boosted trees can be powerful, but they can be harder to explain. Here, a simplistic interpretation is the core requirement, so logistic regression is a better fit.

What the model uses: academic progress (curricular units enrolled/approved across semesters), financial indicators (debtor status, tuition fees up to date, scholarship holder), demographics (gender, marital status, displaced/international status), and context (unemployment rate).

Model Evaluation

Confusion matrix + Metrics

The confusion matrix compares each student‚Äôs true outcome (what happened) to the model‚Äôs predicted outcome. Rows are actual labels; columns are predicted labels. The diagonal shows correct predictions, and off-diagonal cells show mistakes (where one outcome is confused for another).

What it is (and isn‚Äôt) comparing:
This matrix compares outcomes vs predicted outcomes. It does not compare individual features (like age or gender) directly. Features affect predictions, but the matrix only counts whether predictions match the true outcome.

TP / FP / FN / TN and the confusion matrix:
In a multiclass problem like Dropout, Enrolled, and Graduate, these terms are defined one class at a time using a ‚Äúone-vs-rest‚Äù view. For example, when focusing on Dropout, a true positive is a student who actually dropped out and was predicted as Dropout. A false positive is predicted as Dropout but stayed enrolled or graduated, while a false negative is a true dropout predicted as something else. A true negative is a non-dropout correctly predicted as non-dropout. In the confusion matrix, rows represent the true outcomes and columns represent the model‚Äôs predictions. The diagonal cells show correct predictions, while the off-diagonal cells show errors where one outcome is confused for another. Metrics like precision and recall summarize these patterns across the dataset.

Why use a confusion matrix?
Accuracy alone can hide which classes are being confused. The confusion matrix shows the exact error pattern, which is especially important for three outcomes.

Key Metrics

Accuracy: 74.12%
Macro Precision: 71.24%
Macro Recall: 71.40%

These metrics summarize how the model performs across the same test students used in the confusion matrix. Accuracy is overall correctness. Precision penalizes false positives. Recall penalizes false negatives. Macro averages treat each class (Dropout, Enrolled, Graduate) equally.

Math

For any class:
Precision = TP / (TP + FP)
Recall = TP / (TP + FN)

These values are computed for each class using a one-vs-rest approach, then averaged (macro average) to summarize multiclass performance.

Note:

The value of 4,424 is the full dataset size and number of students from the original study. The value of 885 is the number of students were held out as a test set for evaluating the model. All reported metrics and the confusion matrix are based on this test set.Test set size: The evaluation metrics and confusion matrix are computed on a held-out test set of 885 students (20% of the dataset). This helps estimate how well the model generalizes to new students.

Discussion

This project shows how an interpretable model can support education outcomes research. The main value is not just prediction, but a clear view into which factors are linked to dropout risk and student success.

In the real world, a tool like this could help guide supportive interventions (advising, tutoring, financial outreach). Because the data is observational, these results are associations, not causal claims, and fairness checks are important before deployment.
